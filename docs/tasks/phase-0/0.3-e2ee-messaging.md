# 0.3 — E2EE Messaging, Server-Relayed

**Phase:** 0 — Foundation · **Feature:** T03 · **Depends on:** 0.2 · **Status:** done

## Goal
Two CLIs hold an end-to-end-encrypted chat relayed through the server, which sees only opaque
ciphertext — X3DH handshake + Double Ratchet via vodozemac (ADR 0011).

## Scope
- **In:** X3DH, Double Ratchet session, message encrypt/decrypt, `mrd.chat/1` stream, at-rest encryption.
- **Out:** P2P transport (T04), verification/safety-numbers (T08).

## Deliverables
1. `apps/crypto/` (`x3dh.rs`, `ratchet.rs`, `session.rs`, `fingerprint.rs`, `at_rest.rs`).
2. `apps/proto` chat/msg envelope types; `apps/cli` chat demo.

## Risks / notes
Never hand-roll crypto (crypto-protocols skill). Forward secrecy / PCS assertions must not be weakened.

## Tests
`cargo nextest run -p meridian-crypto` · opacity audit `harnesses/opacity-audit/run.sh`

## Reviews
security-reviewer (X3DH/ratchet, FS/PCS), architect (ADR 0011 conformance), test-engineer.
