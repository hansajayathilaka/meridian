# 1.28 — Active relay-rewrite adversarial test (rendezvous `test-tamper-hook` extension)

**Phase:** 1 — Review of Phase 0 · **Feature:** on-the-fly (open coverage gap, not a numbered finding) · **Depends on:** — · **Status:** pending

## Goal
Prove, adversarially, that a malicious/compromised rendezvous *actively rewriting* a routed P2P
signaling blob (not just substituting a `Fetch` bundle) cannot get past DTLS-fingerprint binding — closing
an open note left by `apps/core/src/session.rs`'s module doc and `harnesses/mitm-sim/run.sh`, both of
which currently point at a stale task number ("1.16"/"1.23") for this gap.

## Scope
- **In:** extend `meridian-rendezvous`'s `test-tamper-hook` feature (currently only substitutes `Fetch`
  bundle responses, per 1.12) to also support rewriting a *routed* blob in transit; a harness test driving
  a real `dial`/`answer` P2P session establishment through a tampering rendezvous, asserting the rewrite
  is detected and the session establishment fails closed (fingerprint mismatch), never silently accepts a
  substituted offer/answer.
- **Out:** any production (non-`test-tamper-hook`) code path; this is adversarial test infrastructure
  only, gated the same way as the existing `Fetch`-substitution tamper hook (dev-only feature, never
  enabled in the release `meridian-rendezvous` binary).

## Deliverables
1. `test-tamper-hook` extended to rewrite routed blobs.
2. A harness test (likely alongside `harnesses/mitm-sim/`) proving the rewrite is caught, fail-closed.
3. Doc-sync: repoint `apps/core/src/session.rs`'s module doc and `harnesses/mitm-sim/run.sh`'s stale
   "1.16"/"1.23" references to this task.

## Risks / notes
This is a **security-sensitive server-side change** (even though gated to a dev-only feature never
compiled into the release binary) — route through **security-reviewer** before merge, same standard as
1.12's original tamper hook. Not blocking Phase 1's exit criteria or finding F11 — F11's wire-level
closure is [1.27](./1.27-pcap-assertions-ci.md), which does not depend on this task. Recorded here rather
than silently dropped, per the codebase's convention of tracking every raised gap instead of letting it
evaporate (flagged during the [1.23](./1.23-netns-nat-matrix.md) → 1.24–1.27 split).
