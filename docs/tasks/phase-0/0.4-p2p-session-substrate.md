# 0.4 — P2P Session Substrate

**Phase:** 0 — Foundation · **Feature:** T04 · **Depends on:** 0.3 · **Status:** done

## Goal
Move the chat off the relay onto a direct P2P session (WebRTC data channels behind the `Transport`
trait) so conversations survive a server shutdown; add the stream registry + ctrl/signal wire types.

## Scope
- **In:** `Transport` trait + `LoopbackTransport`, WebRTC data channels, ctrl/signal frames, stream registry.
- **Out:** NAT traversal/TURN (T05), media (T10).

## Deliverables
1. `apps/transport/` (`Transport` trait, loopback), `apps/signaling/`.
2. Stream registry + `stream-types-v1` extension contract; `apps/cli` session demo.

## Risks / notes
Additive stream types touch the registry only — zero core-crate edits (stream-type-authoring skill).
SDP opacity audited (`harnesses/opacity-audit`).

## Tests
`cargo nextest run -p meridian-transport -p meridian-signaling` · mitm-sim + netns two-LAN rig

## Reviews
architect (Transport trait, registry contract), connectivity-debugger, security-reviewer (SDP opacity).
