# 1.12 — Rendezvous fail-closed config + feature-gate the tamper hook

**Phase:** 1 — Review of Phase 0 · **Feature:** review fix (F16, F17) · **Depends on:** — · **Status:** pending

## Goal
The server never silently weakens its posture on a config error, and the bundle-tamper test hook can
never ship in a release binary — closing two threat-model goal 6 ("never silently weaker") violations.

## Scope
- **In:**
  - **F16:** at `apps/rendezvous/src/main.rs:32-38`, exit non-zero when a `--config` path is explicitly
    supplied but fails to parse — today it warns and boots with defaults, silently turning invite-only
    into open registration. (No config supplied → defaults is still fine.)
  - **F17:** gate the bundle-tamper test hook at `apps/rendezvous/src/ws.rs:230-235` behind a cargo
    feature excluded from release builds, instead of a runtime TOML flag present in the production binary.
- **Out:** broader config-schema validation work.

## Deliverables
1. Edited `apps/rendezvous/src/main.rs` — fail-closed on explicit-but-unparseable config.
2. Edited `apps/rendezvous/src/ws.rs` + `apps/rendezvous/Cargo.toml` — tamper hook behind a
   non-default, test-only cargo feature.
3. Tests: (a) bad explicit `--config` path exits non-zero; (b) tamper hook absent without the feature.

## Risks / notes
Distinguish "no config path given" (defaults OK) from "path given but unparseable/missing" (must fail).
The tamper-hook feature must not be in the default feature set and must be excluded from release builds
— confirm `cargo build --release` compiles the hook out entirely.
