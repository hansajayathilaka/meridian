# 1.7 — SecretStore KDF op: drop the signature-determinism dependency

**Phase:** 1 — Review of Phase 0 · **Feature:** review fix (F7) · **Depends on:** — · **Status:** pending

## Goal
The at-rest session-store key no longer silently depends on Ed25519 signature determinism, so a future
HSM/enclave `SecretStore` with randomized signatures can never make all persisted ratchet state
permanently undecryptable.

## Scope
- **In:** add a dedicated key-derivation operation to the `SecretStore` trait
  (`apps/store/src/lib.rs:70-83`) whose output does not rely on signature determinism; migrate
  `apps/crypto/src/at_rest.rs:19-24` off "sign a fixed label" onto the new op; resolve the existing
  `TODO: confirm`; document the derivation as a hard recoverability dependency until the backend contract
  is fully proven.
- **Out:** implementing an actual HSM/enclave backend; changing on-disk format for existing data beyond
  what the migration requires (call out any migration need explicitly).

## Deliverables
1. New derive-key method on the `SecretStore` trait + the loopback/default impl.
2. Edited `apps/crypto/src/at_rest.rs` using the new op.
3. Recoverability note in the store/crypto docs; resolved `TODO: confirm`.
4. Test that the derived store key is stable across calls and independent of signature randomness.

## Risks / notes
This is a trait-contract change — route to **architect**. If it changes the derived key for existing
stores, either preserve backward compatibility or document the migration; do not silently orphan data.
Keep the derivation domain-separated from message/root keys.
