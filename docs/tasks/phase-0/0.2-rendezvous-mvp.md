# 0.2 — Rendezvous Server MVP

**Phase:** 0 — Foundation · **Feature:** T02 · **Depends on:** 0.1 · **Status:** done

## Goal
A single-org rendezvous server where two CLIs register and fetch each other's verified prekey bundles;
the server routes only opaque blobs and never depends on `meridian-core`.

## Scope
- **In:** auth, prekey-bundle publish/fetch, opaque routing, `meridian-proto` wire types.
- **Out:** federation (T06), offline mailbox (T07).

## Deliverables
1. `apps/rendezvous/` (axum + sqlx) with bundle store + opaque relay.
2. `meridian-proto` envelope/bundle types (server's only core dep).

## Risks / notes
Server-no-core invariant enforced by `tools/lint-server-no-core.sh`; no serde on opaque blobs.

## Tests
`cargo nextest run -p meridian-rendezvous` · `just lint-invariants`

## Reviews
architect (dep graph acyclic), security-reviewer (opacity), devops (config surface).
