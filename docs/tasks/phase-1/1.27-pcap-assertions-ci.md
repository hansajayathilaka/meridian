# 1.27 — pcap-analysis assertions + CI/harness wiring (closes F11 wire-level)

**Phase:** 1 — Review of Phase 0 · **Feature:** review fix (F11 wire-level) · **Depends on:** 1.26 · **Status:** pending

## Goal
The pcap captures from [1.26](./1.26-netns-drive-and-capture.md) are turned into strict, fail-closed
pass/fail assertions — this is the task that actually closes F11's wire-level half, not merely "the demo
didn't crash." This is the invariant that makes "relay-only" a real IP-hiding guarantee provable on the
wire, not just in-process (1.16's `enforce_relay_only`) or at the CLI-plumbing level (1.24).

## Scope
- **In:** a pcap-analysis helper (script or small Rust/Python tool) asserting: (a) the negotiated path
  per cell matches the expected rung (direct/srflx/relay-udp/relay-tls-443); (b) under `relay-only`,
  **zero** packets in `ns-bob`'s capture contain alice's host/srflx addresses — any match is a hard
  failure, never a warning; (c) `ns-turn`'s capture contains only DTLS ciphertext, never plaintext (e.g.
  no cleartext SDP/chat bytes visible via a simple string/pattern scan); wire
  `tools/netns-nat-matrix.sh matrix` to run all four cells end-to-end (1.25 topology → 1.26 drive+capture
  → this task's analysis) and exit non-zero on any assertion failure; CI wiring if root+netns is
  available there, else an `--ignored`/harness job gated on root/`NET_ADMIN` with clear SKIP messaging
  and documented manual run instructions, mirroring `tools/netns-nat-matrix.sh`'s existing `need_root()`
  pattern.
- **Out:** any new topology/driving logic (already done in 1.25/1.26); any change to `WebRtcTransport` or
  `enforce_relay_only` — this task proves the existing 1.16 guarantee at the wire level, it does not
  re-implement it.

## Deliverables
1. The pcap-analysis helper with strict (a)-(c) assertions.
2. `bash tools/netns-nat-matrix.sh matrix` passing all four cells end-to-end.
3. CI wiring or a documented gated/manual harness job.

## Risks / notes
Keep the assertions strict per the anonymity-model skill's "must never" list — no downgrading a leak to a
warning. This is the task whose completion actually closes finding F11 (wire-level); 1.14/1.22/1.24/1.25
are all prerequisites, not the closure itself.

## Tests

helper, run standalone against saved pcaps for fast iteration.
