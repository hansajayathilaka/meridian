gap that the entire library-choice rationale (AGPL avoidance) depends on.

## Result
Added `deny.toml` at the repo root encoding ADR 0011/0015's AGPL-avoidance rationale: `[licenses]`
allow list is the exact permissive set actually present in the workspace's dependency graph
(Apache-2.0, Apache-2.0 WITH LLVM-exception, MIT, BSD-1/2/3-Clause, ISC, Unicode-3.0, Zlib,
`confidence-threshold = 0.8`) — no AGPL/GPL/LGPL/copyleft license is allow-listed, so any crate
whose only license option is one of those fails the check. `[bans]` kept light per scope
(`multiple-versions = "warn"`, no strict ban policy). `[advisories]` uses the default RustSec DB
with one documented, narrowly-scoped ignore (`RUSTSEC-2026-0173`, `proc-macro-error2` flagged
unmaintained — not a vulnerability — transitive build-time-only dep via `age` → `i18n-embed-fl`).
`[sources]` restricts to crates.io only.

**F4 closed:** `.github/workflows/ci.yml`'s `supply-chain` job now runs real `cargo deny check` via
`EmbarkStudios/cargo-deny-action@v2` instead of the `echo "TODO cargo-deny check..."` placeholder.

**F18 closed:** clippy is now a blocking gate in both CI (`.github/workflows/ci.yml`'s `lint` job)
and the `Justfile`'s `lint` recipe — `cargo clippy --workspace --all-targets -- -D warnings` with
`|| true` and the `TODO` comments removed. The tree was confirmed clean under the stricter gate, so
flipping it introduced zero new failures (matched the review report's claim).

**AGPL negative-test verification (per the task's Tests requirement):** temporarily added
`quickwit-core = "0.3.0"` (a real crates.io crate, `AGPL-3.0-or-later`, pulling in several more
AGPL-3.0-or-later crates) to `tools/xtask/Cargo.toml`; `cargo deny check licenses` correctly failed,
rejecting each AGPL crate individually. Fully reverted (`Cargo.toml`/`Cargo.lock`); confirmed `git
diff --exit-code` clean afterward and `cargo deny check` passing again against the real workspace.

**Verification:** `cargo deny check` → `advisories ok, bans ok, licenses ok, sources ok`; `cargo
clippy --workspace --all-targets -- -D warnings` → clean; `cargo fmt --all -- --check` → clean;
`cargo build --workspace` → clean. All re-confirmed independently after implementation.

**Reviews:** **devops** PASS (confirmed `cargo-deny-action@v2` is the correct/standard action and
usage, CI job ordering and Justfile/CI parity are sound, no out-of-scope files touched).
**security-reviewer** PASS (confirmed no copyleft license in the allow list, the allow list isn't
overbroad enough to admit one later, the one advisory ignore is honestly scoped and documented,
sources correctly restricted to crates.io; noted it lacked Bash access to run `cargo deny check`
itself, which I independently ran and confirmed clean both before and after the review).
