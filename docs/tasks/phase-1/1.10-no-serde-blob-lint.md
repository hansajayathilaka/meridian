# 1.10 — Harden the no-serde-on-blob lint so it catches the real violation

**Phase:** 1 — Review of Phase 0 · **Feature:** review fix (F15) · **Depends on:** — · **Status:** done

## Goal
The `no-serde-on-blob` guard actually prevents the server from deserializing envelope content — closing
the gap where the current lint matches neither the server's real decode style nor a hypothetical envelope
decode.

## Scope
- **In:** at `tools/lint-no-serde-on-blob.sh:8,12` the lint matches only a literal `payload:` field and
  turbofish deserialization, missing the server's actual type-inferred `frame.decode()` style. Prefer the
  **structural** fix: move envelope-content types into a module the `apps/rendezvous` crate cannot import
  (so the compiler enforces it), or, failing that, deny envelope-content type names from being imported
  into `apps/rendezvous/src`.
- **Out:** the metrics lint (1.9); broader crate-boundary refactors beyond content types.

## Deliverables
1. Hardened `tools/lint-no-serde-on-blob.sh` and/or a module-boundary change in `apps/proto`/`apps/core`.
2. A negative test / lint fixture proving it trips on `frame.decode()`-style inferred deserialization of
   content types in the server crate.

## Risks / notes
Structural enforcement (compiler-level module boundary) is strictly better than a grep — the report
prefers it. Keep the server able to route opaque ciphertext frames; only *content* decode must be
impossible. Confirm the server still builds after any module move.

## Tests
`bash tools/lint-no-serde-on-blob.sh` (must fail on the fixture, pass on the tree) · `cargo build -p meridian-rendezvous`.

## Reviews
**security-reviewer** (server-never-sees-plaintext invariant), **code-reviewer** (module boundary sound).

## Result
**Structural fix (preferred option, delivered):** extracted every content-shaped wire type
(`MessageEnvelope`, `Prekey`, `ChatContent`, `SignalContent`, `CtrlFrame`, and related) out of
`meridian-proto` into a brand-new crate `meridian-envelope` (`apps/envelope/`), which
`meridian-core` depends on and `meridian-rendezvous` does **not** — not in `[dependencies]`, not in
`[dev-dependencies]`, not transitively (verified via `cargo tree -p meridian-rendezvous -e
normal,dev --all-features`, no `meridian-envelope` in the graph). `apps/proto` now exports only
control-plane/opaque-routing types (`OpaqueBlob`, `Frame`, `Op`, `PrekeyBundle`, `Auth`, `Publish`,
`Fetch`, `RouteBody`, `Deliver`, `TurnReq`, etc.) — everything `apps/rendezvous/src/ws.rs`
legitimately `frame.decode()`s. Proven with a temporary probe import
(`use meridian_envelope::MessageEnvelope`) added to `apps/rendezvous/src/lib.rs`: `cargo build
--workspace` failed with `E0432: unresolved import` — a genuine compiler-enforced boundary, immune
to Cargo's workspace feature-unification (which defeated an earlier Cargo-feature-flag attempt).
Probe reverted; full workspace builds/tests/clippy/fmt all clean afterward.

**Lint hardening (defense-in-depth, since a careless future PR could still reach the compiler
boundary from inside a crate that legitimately depends on `meridian-envelope`):**
`tools/lint-no-serde-on-blob.sh`'s check 3 (type-inferred `let x: T = ...frame.decode()` against a
control-plane allowlist) was rewritten from line-based grep to a `perl -0777` whole-file slurp,
closing two review-confirmed bypasses: module-qualified type paths (`content::ChatContent`, now
resolved to their last `::` segment before the allowlist check) and multi-line `let` statements
spanning the type annotation and the `.decode()` call. `--selftest` now covers both bypass fixtures
plus a false-positive check against the real tree, and is wired into `Justfile`'s `lint-invariants`
recipe and `.github/workflows/ci.yml`'s lint job — closing the earlier gap where the self-proving
fixture was a one-time manual artifact never actually run by CI.

**F15 closed:** two independent review rounds. Round 1 found the initial grep-only attempt
insufficient (structural fix untried; two concrete lint bypasses). Round 2, after the rework above,
passed both required reviewers.

**Verification:** `cargo build --workspace`, `cargo test --workspace`, `cargo clippy --workspace
--all-targets [--all-features] -- -D warnings`, `cargo fmt --all -- --check` all clean;
`bash tools/lint-no-serde-on-blob.sh` and `--selftest` both pass; `bash tools/lint-server-no-core.sh`
still passes (unaffected).

**Reviews:** **security-reviewer** PASS (dependency-graph exclusion verified real and complete
including dev-deps; no content-shaped type leaked back into `apps/proto`; both bypasses confirmed
closed; `--selftest` wiring confirmed). **code-reviewer** PASS (independently reproduced both
bypass fixes against the real tree, not just the script's own selftest; spot-checked the new regex
against additional edge cases — `Vec<T>`, nested `Result<Option<T>, E>`, deep module paths — no
further bypass found, one cosmetic-only diagnostic-string quirk on deep nesting; confirmed full
workspace build/test/clippy green).
