# 1.25 — netns topology + NAT-flavor emulation + coturn/rendezvous orchestration

**Phase:** 1 — Review of Phase 0 · **Feature:** review fix (F11 wire, prerequisite) · **Depends on:** 1.14 · **Status:** in progress

## Goal
The six-namespace topology (`ns-alice`/`ns-natA`/`ns-net`/`ns-turn`/`ns-natB`/`ns-bob`) exists and is
provably correct as *networking infrastructure* — real NAT behavior, real coturn, real rendezvous —
before any Meridian process is driven across it ([1.26](./1.26-netns-drive-and-capture.md)'s job).
Independent of [1.24](./1.24-real-signaling-p2p-cli.md); only depends on 1.14 (coturn user-quota), so it
can proceed in parallel with 1.24.

## Scope
- **In:** build the real veth/bridge topology in `tools/netns-nat-matrix.sh` (reusing/extending
  `tools/netns-two-lans.sh`'s `link_pair`/`nat_router` helpers where they fit); apply the per-cell NAT
  flavor on `ns-natA`/`ns-natB` — full-cone (plain MASQUERADE), port-restricted (conntrack-restricted
  reply rules), symmetric (`--random-fully`/per-connection source ports), udp-blocked (drop UDP egress so
  only TCP/TLS survive); launch real coturn in `ns-turn`; launch a real `meridian-rendezvous` instance
  reachable from both `ns-alice`/`ns-bob` so ephemeral HMAC TURN credentials are minted through the real
  `TurnReq`/`TurnGrant` flow (`apps/signaling/src/client.rs::request_turn_credentials`) matching 1.14's
  `user-quota=4` scheme — not a hand-rolled coturn-only credential script; robust `down` teardown covering
  every new resource (iptables rules, coturn/rendezvous processes), safe against a failed run leaving
  stale namespaces behind.
- **Out:** driving any `meridian` peer process, tcpdump capture, pcap analysis, CI wiring (→
  [1.26](./1.26-netns-drive-and-capture.md)/[1.27](./1.27-pcap-assertions-ci.md)).

## Deliverables
1. Topology build/teardown in `tools/netns-nat-matrix.sh`.
2. Per-cell NAT-flavor application (all four cells).
3. Coturn + rendezvous running inside the topology with real ephemeral-credential minting.
4. A generic (non-Meridian) connectivity smoke check per cell proving each NAT flavor and TURN
   reachability behave as intended (e.g. a raw UDP/TCP probe or `turnutils_uclient`/STUN client) — this
   is independently testable without 1.24 landed.

## Risks / notes
Requires root/`NET_ADMIN`; keep the existing `need_root()` graceful-skip pattern from
`tools/netns-nat-matrix.sh`. `TODO: confirm`: exact placement of the rendezvous instance in the topology
(its own namespace vs. living in `ns-net`) — genuinely undecided by existing docs, resolve during
implementation, not by guessing. Re-validate that the coturn config still starts correctly with 1.14's
`user-quota` change in this real topology (previously validated only outside netns).

## Tests
`sudo bash tools/netns-nat-matrix.sh <topology-build subcommand>` for each of the four cells, asserting
build + smoke-check + teardown succeed (root/`NET_ADMIN` required; skips gracefully otherwise, matching
`need_root()`).
