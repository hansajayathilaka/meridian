# 1.5 — Close the two zeroization gaps in meridian-crypto

**Phase:** 1 — Review of Phase 0 · **Feature:** review fix (F5, F6) · **Depends on:** — · **Status:** pending

## Goal
No secret material leaves scope un-zeroized in either the X3DH master-secret assembly or the
`DoubleRatchet` `Drop`, closing the two leaks that negate the surrounding `Zeroizing` discipline.

## Scope
- **In:**
  - **F5:** wrap the concatenated X3DH master secret at `apps/crypto/src/x3dh.rs:94-102,132-140` in
    `Zeroizing<Vec<u8>>` (or equivalent) before deriving the root key — today it is a plain `Vec<u8>`
    dropped without zeroization, negating the per-leg `Zeroizing` wrappers upstream.
  - **F6:** add the four header-encryption keys to `DoubleRatchet`'s `Drop` zeroize set at
    `apps/crypto/src/ratchet.rs:70-84`.
- **Out:** other crates; the broader `SecretStore` KDF change (→ 1.7).

## Deliverables
1. Edited `apps/crypto/src/x3dh.rs` (concatenation buffer wrapped).
2. Edited `apps/crypto/src/ratchet.rs` (`Drop` covers header-encryption keys).
3. Unit test(s) asserting zeroization coverage where observable.

## Risks / notes
Do not change KDF inputs/labels while wrapping the buffer — behavior must be byte-identical (the 1.6
vectors must still pass). `Zeroizing` must wrap the buffer that actually feeds the KDF, not a copy.

## Tests
`cargo nextest run -p meridian-crypto`
