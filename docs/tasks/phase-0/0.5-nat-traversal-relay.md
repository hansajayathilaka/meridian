# 0.5 — NAT Traversal & Relay Policy

**Phase:** 0 — Foundation · **Feature:** T05 · **Depends on:** 0.4 · **Status:** done

## Goal
Establish sessions across symmetric NATs via ephemeral TURN, and add a three-position relay policy where
`relay-only` hides peer IPs; ship a `doctor` connectivity diagnostic.

## Scope
- **In:** ICE/STUN/TURN wiring, ephemeral per-session TURN HMAC creds, relay-only policy, `doctor`.
- **Out:** cross-org federation of TURN (T06/T14), media relay (T10).

## Deliverables
1. NAT traversal + three-position relay policy in `apps/transport`/`apps/cli`.
2. `doctor` diagnostic; NAT matrix rig `harnesses/nat-matrix` + `tools/netns-nat-matrix.sh`.

## Risks / notes
relay-only means "no session" beats "weaker session"; never leak IPs when policy forbids. TURN creds
ephemeral, never in repo.

## Tests
`cargo nextest run -p meridian-transport` · `harnesses/nat-matrix/run.sh`

## Reviews
connectivity-debugger (ICE/NAT edges), security-reviewer (relay-only IP hiding), devops (TURN creds).
